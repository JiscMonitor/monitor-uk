Submodules [hide private]
  • service.api: This module contains all the core system API, for all interactions with the aggregation.
  • service.forms: Module where UI forms are declared
    • service.forms.account: Forms used to provide the user account UI
  • service.lantern: Module which handles all the Lantern integration needs
  • service.models: All core system objects live in this module
    • service.models.account: Core system objects for user accounts
    • service.models.core: All the central APC-focussed model objects
    • service.models.crud: Models associated with the CRUD API
    • service.models.lantern: Model objects to go along with the Lantern integration
    • service.models.workflow: Model objects representing workflow state for record and enhancement processing
  • service.queries: A set of pre-prepared, useful queries, to be applied to the various index types
  • service.scripts: This module contains any scripts that can be run.
    • service.scripts.importer: Importer for Jisc TCO spreadsheets.
  • service.search: Functions and classes used to refine the behaviour of the standard search API
  • service.tasks: Tasks to be executed by the task scheduler
  • service.tests: Unit and functional test suite
    • service.tests.fixtures: Test fixtures for re-use throughout unit and functional tests
      • service.tests.fixtures.lantern: Fixtures for testing Lantern
      • service.tests.fixtures.models: Fixture factories for all the core model objects
    • service.tests.functional: Functional test suite.
      • service.tests.functional.test_crud: Functional tests for the API.
      • service.tests.functional.test_lantern: Functional tests for Lantern integration
      • service.tests.functional.test_search: Functional tests for the Search API
    • service.tests.unit: Unit test suite
      • service.tests.unit.test_api: Unit tests covering the Core API
      • service.tests.unit.test_crud: Unit tests for the CRUD API components
      • service.tests.unit.test_lantern: Unit tests for Lantern components
      • service.tests.unit.test_models: Unit tests for core system models
  • service.views: Blueprints to be bound to the web interface
    • service.views.admin: Admin area view
  • service.web: Main web interface view, pulling together all the blueprints which make up the web component of the application
